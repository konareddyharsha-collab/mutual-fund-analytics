# Mutual Fund Analytics

An end-to-end Extract, Transform, Load (ETL) pipeline and analytics workbench for mutual fund performance evaluation, benchmark tracking, and investor demographics analysis.

## Project Structure

This repository is organized as follows:

*   **[etl/](file:///C:/Users/konar/OneDrive/Desktop/mutual-fund-analytics/etl/)**: The main workspace containing the ETL codebase, sample data, SQLite database, and report generation scripts.
    *   `data/`: Source CSV files containing fund details, NAV history, investor transactions, portfolio holdings, and benchmark indices.
    *   `db/`: Target SQLite database (`mutual_funds.db`) generated by the ETL pipeline.
    *   `schema/`: Database SQL schema definition files.
    *   `notebooks/`: Python scripts for data analysis, metrics calculations, and report generation.
    *   `outputs/`: Generated analytical reports, summaries, interactive Plotly charts, and PDF presentation slide decks.
    *   `requirements.txt`: Python package dependencies.
    *   `README.md`: Detailed configuration and execution instructions for the ETL workspace.

For step-by-step setup, pipeline running commands, and analytical details, please see the **[ETL Workspace README](file:///C:/Users/konar/OneDrive/Desktop/mutual-fund-analytics/etl/README.md)**.

## Key Features

1.  **Automated ETL Pipeline**: Ingests, cleans, and structures mutual fund data into a relational SQLite database with robust mixed-date format parsing.
2.  **Benchmark Comparison**: Computes cumulative relative performance, tracking error (annualized standard deviation of daily return differences vs. Nifty 50), and rolling correlation.
3.  **Investor Demographics**: Performs exploratory data analysis on investor age bands, income distribution, geographic distribution (city tiers), and transaction types.
4.  **Executive Reporting**: Automatically compiles all findings into a clean, 12-slide PDF slide deck ready for stakeholder presentation.

## Resolved Bugs

We resolved several key database schema and metric calculation bugs:
-   **Benchmark Primary Key Overwrite**: Updated the database schema to use a composite primary key on both `(date, index_name)` in the benchmark index table to prevent Nifty 50 and Nifty 100 prices from overwriting each other.
-   **Robust Date Handling**: Fixed date casting to automatically handle both `YYYY-MM-DD` and `DD-MM-YYYY` formats (from local CSVs and API requests) warning-free.
-   **Metric Calculations**: Fixed a tracking error bug that discarded all data when a benchmark was missing, and resolved a rolling correlation NaN issue on smaller datasets.
